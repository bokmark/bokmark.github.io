    RFC 协议官方文档

    网络分层 #

    OSI 分层模型

    ISO 组织讨论的架构，分层更加详细，理想化学、术化，现实中使用TCP\IP模型更多

    graph TD 应用层-->表示层; 表示层-->会话层; 会话层-->传输层; 传输层-->网络层; 网络层-->数据链路层; 数据链路层-->物理层;

    TCP\IP 分层模型

    IEEE 组织开发，更贴近现实

    graph TD; 应用层["应用层：包括OSI模型中的应用层、表示层、传输层"]-->传输层; 传输层-->网络层; 网络层-->D["链路层：包括OSI模型中的数据链路层、物理层"];

    分层说明

    • 应用层：为应用提供服务
    • 表示层（TCP\IP协议中包含在应用层中）：数据格式转化，数据加密
    • 会话层（TCP\IP协议中包含在应用层中）：建立、管理、维护会话
    • 传输层：建立、管理、 维护端对端的连接
    • 网络层：IP地址以及路由选择
    • 数据链路层（TCP\IP协议中包含在数据层中）：提供介质访问和链路管理
    • 物理层（TCP\IP协议中包含在数据层中）：物理传输

    问题： 移动通信的3g、4g在哪个层次？ – 链路层

    TCP\IP协议族模型中有哪些具体协议 #

    包括但不限于

    概念模型 功能 包含的协议
    应用层 文件传输、电子邮件、文件服务、虚拟终端 TFTP、HTTP、FTP、SNMP、SMTP、DNS、Telnet
    应用层 数据格式化、代码转化、数据加密 没有
    应用层 解除或者建立与别的节点的连接 没有
    传输层 提供段对端的接口 TCP、UDP
    网络层 为数据包提供路由 IP、ICMP、RIP、OSPF、BGP、IGMP
    链路层 传输有地址的帧以及错误检测功能 SLIP、CSLIP、PPP、ARP、RARP、MTU
    链路层 以二进制为数据形式在物理媒体上传输数据 ISO2110、IEEE802、IEEE802.2

    TCP\IP协议族中的主要概念 #

    以太网协议中，规定了同一个局域网中的主机与另一个主机通信需要需要知道对方的mac地址。

    传输层主要协议TCP 、 UDP #

    • TCP：面向连接的 可靠的流协议
    • UDP：面向无连接的 不可靠的协议

    IP地址、Mac地址 #

    • ip 在原地址与目标地址之间传递数据包
    • mac地址制造商制作的时候写入，是网络设备的唯一标识
    • 网络层还有icmp 控制报文协议、igmp internet组管理协议等
    • 链路层 arp （地址解析协议）和 rarp（反向地址转化协议）
    • arp 地址解析协议 通过ip转化为mac地址

    端口号 称为程序地址 #

    客户端程序一般情况下不需要端口号，操作系统将会帮忙分配端口号。一般都是分配49152 ~ 65535这些没有人用的端口号

    TCP概述 #

    网络协议为什么要分包 #

    包：全能性术语 帧：数据链路层中包的单位 片：IP数据中的单位 段：TCP数据流中的信息 消息：是指应用协议中的数据和单位

    • 数据链路层对ip层有长度限制：MTU（以以太网为例，MTU=1500字节
    • ip层算上首部 对上层限制：MSS
    • MSS的值一般为MTU值减去两个首部大小（需要减去IP数据包包头的大小20Bytes和TCP数据段的包头20Bytes）

    TCP的基本特性 #

    • 面向连接

      • 只能端对端
      • （udp 既能端对端，也能广播，组播）

    • 可靠性

      • 怎么保证可靠性
        • 超时重传。超过一定时间没有收到应答确认就认为数据包丢掉了，重新发送
        • 应答确认

    • 数据排序

      发送的时候按照顺序发送，接受的时候也有可能不按照顺序接受

      • 物理连接：真实的链路连接
      • 虚拟连接：在三次握手之后也可以认为两端建立了连接 三次握手

      所以tcp数据包有按照数据排序功能

    • 流量控制

      • 滑动窗口： 接受端和发送端有可能存在处理速度不同的问题存在，所以会有一个协商机制存在。在报文的响应中，数据接收方将下次能接受的数据发送给数据发送方。来协商这个发送的数据

    • 全双工 每一端都可以成为数据接受方，也可以成为数据发送方，并且可以同时发送。

    • RTT（往返时延） RTO（重传超时）（动态调整）
