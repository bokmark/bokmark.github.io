# Android 架构
- 业务需求解构， 独立出不可再分的模块
- 再从各个模块中提炼组件
- 组件的不可再发，
- 公用组件下沉
- 模块独立开发，独立测试，明确输入输出，方便编写测试代码
- 搭积木的方式完成整个app的构建
- 使用一些第三方库，来进行组件间的解耦，通信。
- 面向接口编程
- 渐进式的架构
