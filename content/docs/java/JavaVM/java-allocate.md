- 对象的分配策略
  - 对象逃逸分析：分析对象是否可以直接在栈中分配对象，如果是分配对象到栈内存中
  - 本地线程分配缓冲：是否具有本地线程缓冲，如果有直接映射到对应的缓冲中 tlab
  - 是否大对象：如果是大对象直接分配到老年代
  - 对象优先分配到eden区
  - 空间分配担保：虚拟机保证，在分配的时候如果当前的区域有足够的空间分配内存则保证一定那能分配成功，因为会先进行一次gc
  - 大对象直接进入老年代
  - 长期存活的对象进入老年代
  - 对象的gc年龄会动态的判断

- gc年龄
  - 当对象进去eden区之后每次gc都将对象的gc年龄加1。当超过阈值时，将会升级到下一代
  - eden 复制算法 复制到from或者to之中的一个
  - from和to相互复制直到进入老年代
  - 当每个区满的时候将会触发gc
  - minor gc 、young gc 新生代
  - major gc、old gc 老年代
  - full gc 全部区域
  - gc 年龄记在markword 区域
