- jvm 内存结构
  - jmm的规则内存分为线程的工作内存、主内存之分
  - 线程的工作内存包括虚拟栈、本地方法栈、程序计数器
  - 主内存包括堆、方法区
  - 堆按照垃圾回收算法分为 eden、from、to、老年代
  - 方法区分为静态方法区，动态方法区
  - 虚拟机栈是一个栈帧的filo栈。一个栈帧包含操作数栈、本地变量表、动态链接、返回地址
  - 直接内存、本地内存

- gc收集算法有哪些？他们的特点是什么
  - 复制：适用于新生代这种包含大量即用即死的对象情况，标记出不可回收对象之后直接复制到另一个区域。缺点是空间利用率低，只有50%。hotspot的实现是一种变种方式811eden from to的分配比例提高了内存的利用率
  - 标记整理：标记出不可回收对象之后，将对象整理排放。优点是内存规整，下次可以直接使用指针碰撞的方式分配内存，能提高分配的效率。垃圾收集算法的效率不高。
  - 标记清除：标记出不可回收对象之后，将回收对象抹除，优点是回收算法的效率较高。缺点就是大量的内存碎片。下次分配只能使用空闲列表的方式，甚至会导致明明内存足够却无法分配内存的情况存在。


- jvm中一次完整的gc流程是怎么样的？对象如何晋级到老年代？
  - 需要明确gc的算法。
  - 当一个区满的时候进行gc。比如新手代对象满了 进行一次minorgc。老年区满了进行一次major gc。fullgc 就是全局gc
  - 一个正常对象在eden区分配，经过一次gc 将未回收对象复制到from/to区 再经过一次gc复制到to/from区。gc年龄将会记录在对象头的markword区域内，当这个值超过阈值的时候，将会复制到老年代。
  - 新生代的算法都是使用复制算法
  - 当对象进入老年代之后使用复制整理或者复制清楚算法。
  - cms 算法使用复制清除算法。
  - 复制整理算法，将会暂停所有用户线程，然后进行标记整理。
  - cms算法会先进行一次stw标记出和gcroot直接引用的对象。然后并发的进行标记。再进行一次stw重新标记校验。然后再并发的进行清理内存
